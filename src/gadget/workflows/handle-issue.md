---
description: Automatically handles a GitHub issue from branch creation to implementation and AI review.
---

1. Ask the user: "Please provide the GitHub Issue ID or Issue Number you would like me to work on." Wait for their input.
2. Read the issue using the `gh issue view <issue-id>` command.
3. Verify the issue status labels BEFORE proceeding. The issue MUST have the `status: confirmed` label and MUST NOT have the `status: in-progress` label. If these conditions are not met, halt the workflow and inform the user.
4. Determine an appropriate branch name containing the issue identifier (e.g., `issue-<id>-brief-description`).
5. Support independent code development by creating a new worktree and branch, and change into the new directory to isolate your work: `git worktree add ../<branch-name> -b <branch-name> && cd ../<branch-name>`.
6. Invoke the `@research` skill to deeply analyze the issue details and the project context. The skill will output a `research_report.md` artifact with its findings.
7. Invoke the `@design` skill. The true agent will read the `research_report.md` artifact, the entire codebase, project documentation, and user guidelines/rules. It will create a detailed design architecture, saving it as a `design_report.md` artifact without writing actual implementation code.
8. Based on the `research_report.md` and `design_report.md` artifacts, and the issue content, write a detailed implementation plan in **Korean** and add it as a comment to the issue using `gh issue comment <issue-id> -b "<plan>"`.
9. Implement the code changes exactly according to the plan created in the previous step.
10. Once your implementation is ready, invoke the `@review` skill to analyze your changes.
11. Read the review feedback artifact (e.g., `review_report.md`) generated by the reviewing agent and iteratively refactor the code until it meets the required quality standards.
12. If the review artifact contains **no issues and no items requiring user review**, proceed with the following steps automatically:
    1. Push and create a pull request: `git push -u origin HEAD && gh pr create --title "$(git log -1 --pretty=%s)" --body "Resolves #<issue-id>"`
    2. Squash-merge the PR: `gh pr merge --squash --auto`
    3. Clean up the local environment: `cd - && git worktree remove ../<branch-name> && git branch -D <branch-name> && git push origin --delete <branch-name>`
    4. Sync with remote: `git pull`
