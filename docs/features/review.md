# 기능 명세서: review 스킬

## 1. 개요 및 목적
`review` 스킬은 구현이 완료된 코드나 PR을 대상으로 코드 품질, 아키텍처 일관성, 버그 유무, 그리고 프로젝트 규칙 준수 여부를 검토하는 QA(Quality Assurance) 역할을 수행합니다.

## 2. 사용법 (Trigger)
- **명령어**: `@review`
- `handle-issue` 워크플로우 내에서 구현 단계 이후에 호출됩니다.

## 3. 주요 동작 방식 및 내부 로직
1. **변경사항 분석**: 커밋된 코드나 워크트리의 변경된 파일을 `git diff` 등을 통해 분석합니다.
2. **규칙 검증**: `GROUND_RULES.md` 및 `docs/architecture.md` 등의 가이드라인과 코드가 부합하는지 점검합니다. 테스트 코드 포함 여부와 클린 아키텍처 원칙(rule → document → code) 준수 여부를 중요하게 봅니다.
3. **피드백 산출**: 발견된 문제점, 개선 제안, 칭찬 포인트 등을 담은 `review_report.md` 아티팩트를 생성합니다.

## 4. 제약 사항 및 예외 처리
- **직접 수정 금지**: 리뷰어 역할로서 문제점을 지적할 뿐, 직접 코드를 수정해서는 안 됩니다.
- 보고서에 지적된 이슈가 없으면(no issues) 워크플로우의 다음 단계(`pr-create`)로 진행될 수 있습니다.
- 산출물(`review_report.md`)은 한국어로 작성되어야 합니다.
