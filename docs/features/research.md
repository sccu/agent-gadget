# 기능 명세서: research 스킬

## 1. 개요 및 목적
`research` 스킬은 주어진 문제나 요구사항에 대해 코드베이스와 관련 문서를 깊이 있게 탐색하고 분석하여, 아키텍처 이해와 해결책 도출을 위한 기반 자료를 제공하는 역할을 합니다.

## 2. 사용법 (Trigger)
- **명령어**: `@research`

## 3. 주요 동작 방식 및 내부 로직
1. **정보 탐색**: 요구사항과 관련된 파일, 의존성 모듈, 유사한 구현 패턴 등을 코드베이스 내에서 검색(`grep`, `glob` 등)합니다.
2. **구조 분석**: 기존 아키텍처의 제약 사항(`GROUND_RULES.md` 등)과 시스템의 현재 상태를 파악합니다.
3. **보고서 산출**: 발견된 정보와 문제 해결을 위한 사전 분석 내용을 담은 `research_report.md` 아티팩트를 생성합니다.

## 4. 제약 사항 및 예외 처리
- **수정 금지**: 코드를 수정하거나 변경해서는 안 되며, 오직 읽기와 분석만을 수행해야 합니다.
- 분석 범위가 너무 방대할 경우 시스템 과부하를 막기 위해 적절한 단위로 탐색을 제한해야 합니다.
- 산출물은 한국어로 작성되어야 합니다.
