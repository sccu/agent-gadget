# 기능 명세서: design 스킬

## 1. 개요 및 목적
`design` 스킬은 시스템 요구사항과 `research_report.md` 등의 분석 자료를 바탕으로, 구체적이고 실행 가능한 아키텍처 및 컴포넌트 수준의 설계안을 작성하는 역할을 담당합니다.

## 2. 사용법 (Trigger)
- **명령어**: `@design`
- `handle-issue` 워크플로우 내에서 필수적으로 호출됩니다.

## 3. 주요 동작 방식 및 내부 로직
1. **맥락 수집**: `research_report.md`, 현재 코드베이스, 이슈 설명, 프로젝트 규칙(`GROUND_RULES.md`)을 읽어들입니다.
2. **설계안 작성**: 시스템 내에 추가 혹은 수정해야 할 컴포넌트, 데이터 흐름, 의존성, 엣지 케이스 및 구현 단계를 설계합니다.
3. **아티팩트 생성**: 작성된 설계 내용을 `design_report.md`라는 마크다운 파일로 저장합니다.
4. **이슈 코멘트 등록**: 이슈 번호가 제공된 경우, 생성된 보고서를 GitHub 이슈 코멘트로 자동 등록합니다.

## 4. 제약 사항 및 예외 처리
- **구현 코드 작성 금지**: 실제 동작하는 최종 코드를 작성해서는 안 되며, 설계 의도를 보여주기 위한 가상의 의사코드(pseudo-code)나 시그니처만 허용됩니다.
- 산출물(`design_report.md`)은 반드시 한국어로 작성되어야 합니다.
