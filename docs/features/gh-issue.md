# 기능 명세서: gh-issue 스킬

## 1. 개요 및 목적
`gh-issue` 스킬은 사용자의 프롬프트를 기반으로 GitHub CLI(`gh`)를 활용하여 새롭고 체계적인 형태의 GitHub 이슈를 생성하고 관리하는 역할을 합니다.

## 2. 사용법 (Trigger)
- **명령어**: `@gh-issue <prompt>`

## 3. 주요 동작 방식 및 내부 로직
1. **프롬프트 분석**: 사용자 입력으로부터 이슈 제목과 본문에 들어갈 내용을 추출합니다. 추가 맥락이 필요하면 사용자에게 질의합니다.
2. **이슈 초안 작성**: `bug`, `enhancement`, `documentation` 등의 라벨을 부여하고 마크다운 포맷의 초안을 생성합니다.
3. **승인 및 실행**: (수동 모드의 경우) 사용자에게 승인을 받은 후 `gh issue create` 명령어를 실행합니다. 감사 모드(audit mode)에서는 승인 절차를 생략하고 `status: draft`로 이슈를 생성합니다.
4. **상태 검증 및 변경**: `handle-issue` 워크플로우 실행 시, 이슈가 `status: confirmed` 상태인지 확인하고, 작업 시작 시 `status: in-progress` 라벨을 추가합니다.

## 4. 제약 사항 및 예외 처리
- 사용자 승인 없이 임의로 생성 명령을 실행해서는 안 됩니다 (예외: Audit 모드).
- 기술적인 세부 구현 내용보다는 '무엇을 달성할 것인지(WHAT)'에 집중하여 이슈 본문을 작성해야 합니다.
- 산출물(이슈 내용)은 한국어로 작성되어야 합니다.
