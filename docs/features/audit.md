# 기능 명세서: audit 스킬

## 1. 개요 및 목적
`audit` 스킬은 프로젝트의 최상위 문서(예: `GROUND_RULES.md`, `AGENTS.md` 등)와 실제 구현 코드베이스 간의 불일치를 찾아내어 보고하는 기능을 수행합니다. 의존성 방향(rule → document → code)을 준수하는지 점검하는 핵심 도구입니다.

## 2. 사용법 (Trigger)
- **명령어**: `@audit`
- 사용자가 트리거하면 프로젝트 전체 문서와 코드를 스캔하여 검사합니다.

## 3. 주요 동작 방식 및 내부 로직
1. **문서 및 코드 스캔**: `.md` 문서들과 `src/`, `tests/` 등의 코드를 함께 분석합니다.
2. **비교 검증**: 규칙에 명시된 아키텍처, 의존성 제약 사항, 혹은 에이전트 경계 규칙이 실제 코드에 잘 반영되어 있는지 검증합니다.
3. **결과 산출**: 분석이 완료되면 `audit_report.md` 형태의 결과물 아티팩트를 생성합니다. 이 결과물은 후속 조치를 위해 다수의 GitHub Issue로 변환될 수 있습니다.

## 4. 제약 사항 및 예외 처리
- **수정 금지**: `audit` 스킬은 절대로 코드를 직접 수정해서는 안 됩니다. 역할의 범위를 벗어난 행동은 금지되어 있습니다.
- 산출물은 반드시 한국어로 작성되어야 합니다.
