# 기능 명세서: pr-create 스킬

## 1. 개요 및 목적
`pr-create` 스킬은 코드 구현 및 검증이 완료된 브랜치를 기반으로 GitHub에 Pull Request를 생성하고, 필요한 경우 자동 머지 및 워크트리 정리까지 수행하는 마무리 역할을 담당합니다.

## 2. 사용법 (Trigger)
- **명령어**: `@pr-create <issue-id>`
- 주로 `handle-issue` 워크플로우의 마지막 단계에서 호출됩니다.

## 3. 주요 동작 방식 및 내부 로직
1. **커밋 및 푸시**: 구현된 변경사항을 스테이징, 커밋하고 원격 리포지토리에 푸시합니다.
2. **PR 생성**: GitHub CLI(`gh`)를 활용하여 해당 브랜치의 내용으로 PR을 생성합니다. 관련 이슈를 닫는 키워드(예: `Fixes #<issue-id>`)를 본문에 포함시킵니다.
3. **머지 및 정리**: (옵션 또는 설정에 따라) PR을 자동 머지하고, `.worktrees/`의 작업 디렉토리를 삭제한 뒤 로컬 및 원격 브랜치를 정리합니다.

## 4. 제약 사항 및 예외 처리
- 로컬 및 원격 저장소에 올바른 권한(Token)이 필요합니다.
- 코드가 완전히 검증되지 않은 상태에서는 호출하지 않는 것을 권장합니다.
