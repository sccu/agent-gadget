# 제품 요구사항 정의서 (PRD)

## 1. 개요 및 목적
**agent-gadget**은 에이전트 중심의 개발 프레임워크입니다. 이 프로젝트는 `rule → document → code`라는 엄격한 의존성 흐름을 따르며, 코드보다는 상위 수준의 규칙과 문서 개선을 통해 문제의 근본 원인을 해결하는 것을 목표로 합니다.
본 PRD는 `agent-gadget`의 전체적인 제품 요구사항, 사용자 시나리오 및 주요 기능 요구사항을 정의합니다.

## 2. 사용자 역할 (User Roles) 및 시나리오
- **개발자 (Developer)**: 시스템에 개입하여 에이전트가 처리할 이슈를 제공하고, 최종 PR을 리뷰 및 머지합니다.
- **AI 에이전트 (AI Agent)**: 제공된 이슈를 바탕으로 자동으로 작업 트리를 격리하고, 코드를 분석, 설계, 구현, 리뷰한 뒤 PR을 생성하는 과정을 자율적으로 수행합니다.

### 2.1 주요 시나리오
1. **이슈 생성**: 사용자가 프롬프트를 입력하면 에이전트가 GitHub 이슈를 생성하고 관련 라벨을 부여합니다.
2. **이슈 자동 처리**: 사용자가 특정 이슈 번호를 제공하면 에이전트가 작업을 격리된 환경에서 분석, 설계, 구현하고 PR을 생성합니다.

## 3. 주요 기능 요구사항
### 3.1 에이전트 스킬 (Agent Skills)
- **gh-issue**: GitHub CLI를 사용하여 이슈를 자동으로 생성합니다.
- **research**: 코드베이스를 심층 분석하여 `research_report.md`를 산출합니다.
- **design**: 시스템 아키텍처 및 상세 설계를 수행하여 `design_report.md`를 산출합니다.
- **review**: 구현된 코드의 품질을 검토하고 `review_report.md`를 산출합니다.
- **audit**: 프로젝트 문서와 코드의 불일치를 찾아내어 `audit_report.md`를 산출합니다.
- **nobot**: 봇 탐지를 우회하는 사람과 유사한 브라우저 자동화를 지원합니다.
- **pr-create**: 작업 완료 후 자동으로 PR을 생성하고 머지 및 워크트리 정리를 수행합니다.

### 3.2 워크플로우 (Workflows)
- **handle-issue**: 이슈 접수부터 PR 생성까지의 전체 생명주기를 자동화합니다. 격리된 Git 워크트리를 사용하여 메인 환경의 오염을 방지합니다.

## 4. 비기능적 요구사항
- **의존성 방향**: 반드시 규칙(Rules)에서 문서(Docs)로, 그리고 코드(Code) 방향으로 의존성이 흘러야 합니다.
- **테스트**: 모든 코드는 `pytest` 및 `playwright` 기반의 테스트를 거쳐야 하며, 커밋 전에 훅(pre-commit)을 통과해야 합니다.
- **클린 아키텍처**: 정책(Policy) 함수가 메커니즘(Mechanism) 함수를 호출하는 구조를 유지해야 합니다.
- **환경 격리**: 자동화된 이슈 처리는 `.worktrees/` 하위의 격리된 환경에서 수행되어야 합니다.
